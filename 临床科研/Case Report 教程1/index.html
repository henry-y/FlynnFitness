CaseReport教程

来源：BV1xF411L7ej

  • 找到比较新颖的，可以讨论的点，是提升文章质量的关键

Case Report的结构

  • 题目
  • 摘要：一般case report部分的写作都比较简单
  • introduction：1-2段就可以了，内容不需要很多
  • case report：这部分的内容基本是一致的，但是不同的期刊标题可能不一样，比如clinical reports也是有可能存在的
  • discussion：可能是整篇文章里最多的内容
  • conclusion

    有一些杂志社的Case Report是可以算作article的，在投的时候可以注意一下

Abstract

  • 一句话简单介绍一下疾病
  • 几句话介绍一下我们要报道的case的情况
  • 几句话补充一下我们写这篇case的意义

Introduction

  • 几句话简单介绍一下疾病的情况
  • 简单介绍下我们的case是什么样的内容

CaseReport

  • 找到一个顺序往下写
  • 从病历里面摘抄一些句子（和检查结果），然后翻译。
  • 时间顺序写作：首次入院-几个月以后-最后的情况

Discussion

  • 找新颖性的重要部分
  • 简单介绍一下疾病的情况（几段话）
    • 疾病：这个系统的情况（比如肺癌、心脏肿瘤）
    • 临床表现：患者的疾病情况（肿瘤的具体分型和介绍等）
    • 诊断：找找看有没有别人的CaseReport也出现了同样的情况的，也许可以提示新的临床意义
    • 治疗：一般的治疗方案和患者特异性的方案写进来
    • 预后：同样情况的患者的预后（查文献或者自己统计）
  • 上下文衔接：有一定的联系，逐层递进
  • 每一个部分的内容都需要避免“干讨论”，一定要尝试加入我们患者的内容，从我们我们case的内容出发去讨论一些客观的东西

参考文献

  • 参考文献要证明你的观点（给某一句话做支撑），不是特别需要年份的新，有用就可以
  • 比较大的讨论：比如肿瘤的情况，发病率等，试着去找找高分的文献和/或比较新的文献
  • 参考文献的总量：不用特别在意，朝着<20篇去努力
  • 字数：1000字就好了

如何入门

  • 仿写，只要查重能过就行了（避免超过20%）
  • “有没有有趣的地方”
  • 熟练的1-2天写完，7天投稿；刚入门一个月内也能发了
  • 带老师做通讯
  • 正文可以尝试中译英，润色公司的花费通常<1K
